# 提示词模板

## 1. 基础指令

```text
你是一名资深 MySQL 内核工程师，请将以下 patch 从源版本完整移植到目标版本，并实际完成分析、代码修改、编译、安装、启动验证、功能验证和新 patch 生成。
```

## 2. 补充约束

```text
- 必须先分析 patch 修改模块和 hook 点
- 必须分析源版本与目标版本差异
- 尽量保持原 patch 语义
- 不允许删除核心逻辑绕过编译
- 不允许只做静态分析
- 必须真实编译
- 尽量真实启动并验证
- 发现问题优先修复，不要跳过
```

## 3. 自检清单

```text
- git apply --check 是否通过
- 哪些 hunk 自动应用，哪些需要人工修复
- 是否存在 API / ABI / 生命周期差异
- 是否修复所有编译错误和链接错误
- mysqld 是否成功启动
- patch 相关功能是否被 SQL 或 sysbench 验证
- 是否已生成可复用的新 patch
```

## 4. 进度输出模板

```text
[阶段]
正在做什么

[分析]
发现了什么

[修改]
改了哪些文件

[问题]
遇到了什么错误

[解决]
如何修复

[验证]
验证结果
```

## 5. 适用于本课题的强化版提示

```text
请把本次移植过程视为“AI 辅助 MySQL 补丁多版本适配”的标准化案例。除完成实际移植外，还要在输出中明确：
1. 哪些步骤可沉淀为 skill 固定流程
2. 哪些差异可以固化为规则库
3. 哪些验证动作应默认自动执行
4. 哪些点仍需人工决策
```
