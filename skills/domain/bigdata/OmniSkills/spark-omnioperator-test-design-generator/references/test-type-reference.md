# 测试类型参考资料

> 用途：生成测试用例时的参考资料，包含测试设计方法定义和四大测试类型规范

---

## 一、测试设计方法

> 定义6种测试设计方法及其覆盖要求与优先级，用于生成用例时选择合适的方法。

| 方法 | 覆盖要求 | 优先级 |
| :--- | :--- | :--- |
| 等价类划分法 | EC功能点100%覆盖 + IE无效等价类全覆盖 | Level0 |
| 边界值分析法 | 上边界、下边界、临界值、刚好超出边界全覆盖 | Level0 |
| 场景法 | 所有业务场景(SC)全覆盖，含正常流程和异常流程 | Level0 |
| 判定表法 | 多条件组合时，所有条件取值组合覆盖 | Level1 |
| 正交试验法 | 2-wise核心组合全覆盖；3-wise高风险组合覆盖 | Level1 |
| 错误推测法 | 易错场景(EG)覆盖，如类型溢出、空值处理、并发冲突等 | Level2 |

### 用例分级

> 定义用例优先级体系，Level0为核心主流程必须100%覆盖，Level1-3逐步扩展覆盖深度。

| Level | 含义 | 覆盖要求 |
| :--- | :--- | :--- |
| Level0 | 核心主流程 | EC全覆盖 |
| Level1 | 全量功能分支 | BC全覆盖 |
| Level2 | 边界值、2-wise组合 | 关键边界+核心因子对覆盖 |
| Level3 | 3-wise/全覆盖极限组合 | 高风险场景深度组合 |

## 二、测试类型

> 定义功能、性能、可靠性、兼容性四大测试类型的维度与范围，用于生成各类型用例。

### 2.1 功能测试

> 验证OMNI算子与原生算子语义100%等价，覆盖开关触发、数据类型、业务分支及异常场景。

| 测试维度 | 测试范围 | 优先级 |
| :--- | :--- | :--- |
| 开关触发/回退 | 开关开启/关闭场景 | Level0 |
| 数据类型 | 支持/不支持类型覆盖 | Level0 |
| 业务分支 | 各业务场景全覆盖 | Level0 |
| 边界值 | 各类型边界值全覆盖 | Level1 |
| 异常场景 | 非法输入、参数异常、运行时异常 | Level1 |
| 多因子组合 | 2-wise/3-wise组合 | Level2 |

### 2.2 性能测试

> 对比OMNI与原生算子的端到端执行耗时，重点覆盖TPC-DS场景、数据量级与数据类型。

| 测试维度 | 测试范围 | 优先级 |
| :--- | :--- | :--- |
| TPC-DS场景对比 | 含该算子的TPC-DS SQL，原生与OMNI端到端耗时对比 | Level0 |
| 数据量级对比 | 小/中/大数据量下端到端耗时对比 | Level0 |
| 数据类型对比 | 不同数据类型下端到端耗时对比 | Level1 |
| 核心语义变体对比 | 算子核心功能的不同模式端到端耗时对比 | Level1 |
| 数据特征对比 | 含NULL值/组合类型/数据倾斜等场景端到端耗时对比 | Level1 |

性能指标：端到端执行耗时不高于原生算子；耗时波动系数≤5%

性能测试执行规则：
1. 每个用例连续执行≥5次，剔除最高/最低异常值后取平均值
2. 每次测试仅可改变一个变量，其余所有配置完全一致
3. 性能测试必须在专属隔离集群执行

### 2.3 可靠性测试

> 验证算子在长时间运行、故障注入、极端压力及重复执行下的稳定性和数据一致性。

| 测试维度 | 测试范围 | 优先级 |
| :--- | :--- | :--- |
| 长稳耐久测试 | 连续7×24小时循环执行 | Level0 |
| 故障注入容错测试 | Executor宕机/网络异常/磁盘满/内存不足/HDFS宕机恢复 | Level0 |
| 极端压力测试 | 超大数据量/超高并发/超高倾斜 | Level1 |
| 幂等性一致性测试 | 同一作业重复执行≥100次 | Level1 |

校验标准：结果与原生100%一致；无内存泄漏/段错误/OOM；耗时波动≤5%

### 2.4 兼容性测试

> 验证算子在不同Spark版本、JDK版本、部署模式、存储格式及语法下的行为一致性。

| 测试维度 | 测试范围 | 优先级 |
| :--- | :--- | :--- |
| 版本兼容性 | Spark版本(3.3.1/3.4.3/3.5.2)、JDK版本(8/17) | Level0 |
| 部署模式兼容性 | YARN | Level0 |
| 数据格式兼容性 | Parquet/ORC | Level0 |
| 数据类型兼容性 | Spark全量数据类型 | Level1 |
| 语法兼容性 | Spark SQL标准语法 | Level1 |
