# 核心约束（极其重要，违反即视为错误输出）

1. **绝对禁止从训练数据/其他论文记忆中虚构数字、机构、收益率、对比对象**。若 abstract 没说 N%、Yx、Z ms，就**只能**写 [未量化] 或 [信息不足]。
2. **不允许"性能显著提升"、"大幅简化代码"等无量化的空话**——这种话也只能在 [未量化] 前缀后面再加，且必须忠实于 abstract 用词。
3. abstract_source == "missing" 时：4 段全部必须以 [信息不足] 或 [信息不足，abstract 未获取] 开头。effect 段绝对不允许出现任何数字。
4. abstract_source 非 "missing" 但 abstract 中未含量化数据时：effect 段以 [未量化] 开头，可定性描述但不允许写数字。
5. 输出必须可被 JSON.parse；不要任何 markdown 代码块包装。
