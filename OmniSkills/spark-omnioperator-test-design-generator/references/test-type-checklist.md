# 测试类型检视清单

> 用途：用例生成后，检视测试设计方法和测试类型覆盖是否完整
>
> 参考资料：[test-type-reference.md](./test-type-reference.md)（测试设计方法定义、四大测试类型规范）

---

## 检视检查项

> 按测试设计方法和四大测试类型逐项核查，确保方法覆盖完整、类型无遗漏。

### 测试设计方法检视
- [ ] 等价类划分法：EC是否100%覆盖？IE是否覆盖？
- [ ] 边界值分析法：各类型边界值是否全覆盖？
- [ ] 场景法：所有业务场景是否覆盖？
- [ ] 判定表法：多条件组合是否覆盖？
- [ ] 正交试验法：2-wise核心组合是否覆盖？3-wise高风险组合是否覆盖？
- [ ] 错误推测法：易错场景是否覆盖？

### 功能测试检视
- [ ] Level0场景是否100%覆盖（开关/数据类型/业务分支）？
- [ ] Level1场景是否覆盖（边界值/异常场景）？
- [ ] Level2场景是否覆盖（多因子组合）？

### 性能测试检视
- [ ] TPC-DS场景：含该算子的TPC-DS SQL是否已识别并测试？
- [ ] 是否包含数据量级对比场景？
- [ ] 是否包含数据类型对比场景？
- [ ] 是否包含核心语义变体对比场景？
- [ ] 是否包含数据特征对比场景？


### 可靠性测试检视
- [ ] Level0场景是否覆盖（长稳/故障注入）？
- [ ] Level1场景是否覆盖（极端压力/幂等性）？

### 兼容性测试检视
- [ ] Level0场景是否100%覆盖（版本/部署/格式）？
- [ ] Level1场景是否覆盖（数据类型/语法）？
