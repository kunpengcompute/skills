ARM / aarch64 Platform Specific Notes

  • NEON Coverage Gap: Many x86 SIMD (AVX/SSE) optimizations in native engine lack aarch64 NEON equivalents; string processing and float arithmetic show significant performance gaps.
  • Hash Table Latency: HashJoin/HashAggregate hash tables use same 64B cache line size as x86, but ARM memory access latency differences cause hash probe performance degradation.
  • DWRF/ORC Decode: RLE/BIT_PACK encoding lacks NEON optimization on ARM; decode speed is ~60-70% of x86 equivalent.
  • Compile Flags: Recommended: -march=armv8.2-a+crypto+dotprod for Kunpeng-specific instructions (CRC32, AES, dot-product for hash and matrix ops).
  • Memory Allocator: Consider jemalloc with large page (2MB) support on ARM for better TLB performance and reduced page fault overhead.
