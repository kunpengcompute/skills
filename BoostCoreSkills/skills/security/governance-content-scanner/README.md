# governance-content-scanner

`governance-content-scanner` 是一个用于扫描网页开源治理内容的 Agent skill。它检测网页中是否包含 CLA、SIG 等开源项目治理要素。

## 主要能力

- **CLA 检测**：
  - 识别 Contributor License Agreement、DCO (Developer Certificate of Origin)
  - 区分个人CLA (ICLA)、企业CLA (CCLA)、员工CLA签署类型
  - 检测CLA条款完整性（著作权授予、专利权授予、原创保证、免责条款等）
  - 验证是否声明项目采用的开源协议
- **SIG 检测**：识别 Special Interest Group、Working Group、SIG 组织结构等
- **Code of Conduct 检测**：
  - 检测行为准则位置是否显著且易于查找阅读
  - 检测是否明确适用的行为范围
  - 检测是否明确适用的参与者成员范围
  - 检测是否明确违规行为如何处罚
  - 检测是否有合理的举报违规行为的方式
  - 检测是否提供违反准则者的约束措施（如参与者公约1.4或2.0版本）
  - 检测社区是否积极对举报行为进行回应
  - 检测是否对处理结果进行公示
- **社区多样性检测**：
  - 检测贡献者地域多样性（来自不同国家/地区）
  - 检测贡献者组织多样性（来自不同组织/公司）
  - 评估贡献集中度（主要贡献者贡献占比）
  - 评估新贡献者比例
  - 分析个人与企业贡献者比例
- **其他治理要素**：检测 Contributing Guidelines（贡献指南）
- **链接分析**：检测治理文档相关链接
- **成熟度评估**：对项目治理完善程度进行评分

## 目录说明

- `SKILL.md`：skill 的主说明，包含检测规则、工作流程和报告格式

## 快速使用

**扫描单个页面**

```
prompt: /governance-content-scanner 扫描 https://kubernetes.io/community/ 的治理内容
```

**扫描项目仓库**

```
prompt: /governance-content-scanner 检查 https://github.com/apache/kubernetes 的治理结构
```

**批量扫描**

```
prompt: /governance-content-scanner 扫描以下页面的治理内容：
- https://openeuler.org/zh/community/
- https://www.openeuler.org/zh/community/committee.html
```

## 检测范围

### CLA 检测详情

| 检测项 | 说明 | 必要性 |
|--------|------|--------|
| 签署主体信息 | 明确签署者身份信息 | 必需 |
| 著作权授予 | 授予著作权给项目拥有者 | 必需 |
| 专利权授予 | 授予专利权给项目 | 必需 |
| 授权保证 | 签署者保证有权授予许可 | 必需 |
| 原创保证 | 保证贡献为原创作品 | 必需 |
| 免责条款 | 贡献内容免责声明 | 必需 |
| 非原创处理 | 提交非原创作品的处理方法 | 推荐 |
| 通知义务 | 不准确事实的通知义务 | 推荐 |
| 开源协议声明 | 项目采用的开源协议 | 必需 |

### Code of Conduct 检测详情

| 检测项 | 说明 | 必要性 |
|--------|------|--------|
| 位置显著 | 行为准则文件位于根目录或.github目录，易于查找阅读 | 必需 |
| 行为范围 | 明确适用的行为范围（期望行为和不当行为） | 必需 |
| 参与者范围 | 明确适用的参与者成员范围 | 必需 |
| 违规处罚 | 明确违规行为如何处罚 | 必需 |
| 举报方式 | 有合理的举报违规行为的方式 | 必需 |
| 约束措施 | 提供违反准则者的约束措施（如禁止、警告、封禁等） | 必需 |
| 回应机制 | 社区应积极对举报行为进行回应，处理违规行为 | 必需 |
| 结果公示 | 对处理结果进行公示 | 推荐 |
| 标准版本 | 使用参与者公约1.4或2.0版本等标准版本 | 推荐 |

### 社区多样性检测详情

| 检测项 | 说明 | 评估标准 |
|--------|------|----------|
| 贡献者国家数量 | 统计贡献者所在国家数量 | ≥5个国家为优秀 |
| 参与组织数量 | 统计贡献者所属组织数量 | ≥3个组织为优秀 |
| 贡献集中度 | 主要贡献者贡献占比 | ≤50%为健康 |
| 新贡献者比例 | 新贡献者占比 | ≥20%为活跃 |
| 个人与企业比例 | 个人与企业贡献者比例 | 比例合理为健康 |

### 其他治理要素

| 类别 | 检测内容 | 典型关键词 |
|------|----------|------------|
| 贡献指南 | Contributing Guidelines | `CONTRIBUTING`, `How to Contribute`, `贡献指南`, `贡献路径` |

## 扫描工作流程

```
Step 1: 获取网页内容
    ↓
Step 2: CLA 检测
    ↓
Step 3: SIG 检测
    ↓
Step 4: Code of Conduct 检测
    ↓
Step 5: 社区多样性检测
    ↓
Step 6: 其他治理要素检测
    ↓
Step 7: 链接分析
    ↓
Step 8: 生成报告
```

## 报告内容

扫描报告包含：
- **检测结果摘要**：各治理要素的检测状态
- **详细发现**：每个要素的具体内容和位置
- **治理成熟度评估**：多维度评分和总体评分
- **改进建议**：针对缺失要素的建议

## 常见开源社区治理模式

| 社区 | 治理特点 |
|------|----------|
| Kubernetes | SIG 组织结构、KEPC 提案流程、Contributor Ladder |
| Apache | PMC 治理、Apache Way、投票机制 |
| CNCF | TOC、项目生命周期 (Sandbox/Incubating/Graduated) |
| openEuler | 技术委员会、SIG 组、社区选举 |
