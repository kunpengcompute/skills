## ① 当天报告（一份，含全部类型）

路径：`<标签>/daily/<YYYY-MM>/<YYYY-MM-DD>.md`


**写作约束**：
- 资讯维度三段式（背景/要点/启发）不可省。
- 放宽子窗的 `###` 小节标题标实际天数（`📰 资讯（近 7 天）`）。
- 某类型某维度无内容写一句"无强相关"，**保留小节**（让读者知道你查过了），不删不硬凑。
- 某类型整体清淡可在该 `##` 段开头一句说明；但不要整段省略——保留类型骨架。
- 目录锚点用 GitHub 风格（中文标题锚点为小写、空格转 `-`，emoji 不进锚点）。

**索引维护要点**：
- README **增量更新**：读旧 README → 顶部插入新期号 → 写回。不要重写丢历史。
- 根 README「最近期号」只留最近 ~10 期。

## 反例（别这么写）

- ❌ 把 commit hash 当标题（`abc1234 fix`）→ 读者要"发生了什么"。
- ❌ 只有标题没有要点的资讯条目。
- ❌ 论文抓不到摘要却写"提升 23%"→ 编造，必删。
- ❌ 把某类型整段省略（因为今天它薄）→ 保留类型骨架 + 写"无强相关"，证明你查过。
- ❌ 重写 README 丢掉历史期号 → 必须增量插入。
- ❌ 再去拆 `domains/` 或单独产"合订本"→ 现在就一份报告，类型是 `##` 段不是单独文件。
